# mimo-v2.5 系列模型定价与配置更新

**日期**: 2026-05-30
**状态**: 待审核
**范围**: 更新 mimo-v2.5-pro 和 mimo-v2.5 的模型配置和定价，支持分段计费和缓存命中价格

---

## 背景

mimo-v2.5 系列模型于 2026-04-23 发布，具有 1M 超长上下文和分段定价（按 256K/1M 分段），同时支持缓存命中折扣价格。当前代码的 `CostTracker` 使用固定 input/output 价格，不支持分段和缓存价格。

## 设计目标

1. 扩展 `ModelPricing` 接口支持按上下文长度分段计费
2. 增加缓存命中价格支持
3. 更新 mimo-v2.5-pro 和 mimo-v2.5 的上下文窗口为 1M
4. 删除 mimo-v2-flash，不添加 mimo-v2.5-TTS（TTS 后续单独处理）
5. 不影响其他模型（Claude、GPT 等）的适配

---

## 接口设计

### ModelPricing 接口（新）

```ts
interface ContextTier {
  threshold: number;            // 上下文上限 (tokens)
  inputPricePer1M: number;      // 输入价格 $/1M tokens
  cacheInputPricePer1M: number; // 缓存命中价格 $/1M tokens
  outputPricePer1M: number;     // 输出价格 $/1M tokens
}

interface ModelPricing {
  tiers: ContextTier[];         // 按 threshold 从小到大排列
}
```

### CostTracker 接口变更

```ts
// 新增可选参数 contextTokens（默认 0，匹配最低 tier）和 cacheHit（默认 false）
calculateCost(model: string, usage: TokenUsage, contextTokens?: number, cacheHit?: boolean): number
record(model: string, usage: TokenUsage, contextTokens?: number, cacheHit?: boolean): number

// CostEntry 新增字段
interface CostEntry {
  model: string;
  inputTokens: number;
  outputTokens: number;
  contextTokens: number;  // 新增
  cacheHit: boolean;       // 新增：是否命中缓存
  cost: number;
  timestamp: number;
}
```

Tier 选择逻辑：找到第一个 `threshold >= contextTokens` 的 tier；如果 contextTokens 超过所有 threshold，使用最后一个 tier。

缓存价格逻辑：当 `cacheHit === true` 时，使用 `tier.cacheInputPricePer1M` 计算输入费用；否则使用 `tier.inputPricePer1M`。

## 定价数据

### mimo-v2.5-pro

| 上下文范围 | 输入 ($/1M) | 缓存命中 ($/1M) | 输出 ($/1M) |
|-----------|------------|----------------|------------|
| ≤ 256K    | $1.00      | $0.20          | $3.00      |
| ≤ 1M      | $2.00      | $0.40          | $6.00      |

### mimo-v2.5

| 上下文范围 | 输入 ($/1M) | 缓存命中 ($/1M) | 输出 ($/1M) |
|-----------|------------|----------------|------------|
| ≤ 256K    | $0.40      | $0.08          | $2.00      |
| ≤ 1M      | $0.80      | $0.16          | $4.00      |

### 其他模型（保持不变，单 tier）

| 模型 | 输入 ($/1M) | 缓存命中 ($/1M) | 输出 ($/1M) |
|------|------------|----------------|------------|
| gpt-4o | $2.50 | $2.50 | $10.00 |
| claude-sonnet | $3.00 | $3.00 | $15.00 |

缓存命中价格设为与输入价格相同（表示无缓存折扣），保持向后兼容。

## 测试策略

1. **Tier 选择测试**：验证 200K 上下文匹配 ≤256K tier，500K 上下文匹配 ≤1M tier
2. **缓存价格测试**：验证 cacheInputPricePer1M 被正确使用
3. **向后兼容测试**：不传 contextTokens 时默认使用最低 tier
4. **超限测试**：contextTokens 超过最大 threshold 时使用最高 tier
5. **MIMO_MODELS 测试**：确认只有两个模型，fallback 为 256000
