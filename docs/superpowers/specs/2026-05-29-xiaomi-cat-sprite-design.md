# 小米猫精灵设计文档

**日期：** 2026-05-29
**状态：** 已批准
**范围：** 将现有 buddy 系统的 companion 替换为小米猫精灵（唯一物种）

---

## 1. 设计目标

基于小米公司 logo 的视觉元素和品牌特性，设计一个具有灵动特质的小猫精灵形象，作为 buddy 系统的唯一 companion 物种，替换现有所有动物精灵。

**核心要求：**
- 外观融合小米 logo 标志性元素（MI 字母结构、超椭圆轮廓、橙色主色调）
- 具备流畅自然的动画效果（表情变化、动作响应、状态转换）
- 交互逻辑符合用户直觉，提供即时反馈
- 体现科技感与亲和力的平衡，符合小米品牌年轻化、智能化定位

## 2. 视觉设计

### 2.1 核心概念：猫 = MI 解构

猫的每个身体部位对应 MI 字母的结构：

```
M = 猫耳 — 两个拱形 ╱╲ ╱╲，中间 V 是两耳间的凹陷
I = 猫尾 — 竖线 │，摇动时变为 ╱ 或 ╲
超椭圆 = 猫脸 — ╭╮╰╯ 圆角括号模拟小米 2021 redesign 的"生命感"曲线
```

### 2.2 基础造型

```
   ╱╲    ╱╲        ← M 的两个拱形 = 猫耳
  ╱  ╲──╱  ╲       ← M 的中间 V = 耳间凹陷
  │  ●    ● │      ← 眼睛（● 实心圆）
  │    ω    │      ← 猫嘴（ω）
   ╲      ╱        ← 超椭圆脸下半部
    ╰────╯│        ← 超椭圆底部 + 身体
           │        ← I 的竖线 = 猫尾
```

### 2.3 尺寸约束

- **宽度：** 12字符（与现有精灵一致）
- **高度：** 7行（比现有精灵高2行，耳朵更突出）
- 需检查 FullscreenLayout 的 bottom slot 是否能容纳增加的高度

### 2.4 配色方案

固定小米橙，不随稀有度变化——强化品牌统一性。

| 元素 | 颜色 | 色值 |
|------|------|------|
| 主色调 | 小米橙 | `#FF6900` |
| 耳朵高光 | 浅橙 | `#FF8C33` |
| 眼睛 | 纯白 | `#FFFFFF` |
| 尾巴尖 | 深橙 | `#CC5500` |

**稀有度系统：** 移除。所有用户获得相同的小米猫精灵，外观一致，无稀有度变体。

### 2.5 眼睛样式

精灵的眼睛固定为 `●`（实心圆），不使用现有系统的 `{E}` 占位符替换机制。这是因为小米猫的造型需要与 MI 结构保持视觉一致性。

## 3. 动画系统

### 3.1 闲置动画（循环）

3帧循环，500ms/帧（与现有 TICK_MS 一致）：

**帧0：休息**
```
   ╱╲    ╱╲
  ╱  ╲──╱  ╲
  │  ●    ● │
  │    ω    │
   ╲      ╱
    ╰────╯│
           │
```

**帧1：左顾**
```
   ╱╲    ╱╲
  ╱  ╲──╱  ╲
  │ ●     ● │
  │    ω    │
   ╲      ╱
    ╰────╯ │
           ╱
```
头微左转，尾巴右摆。

**帧2：右顾**
```
   ╱╲    ╱╲
  ╱  ╲──╱  ╲
  │  ●    ●  │
  │    ω    │
   ╲      ╱
    ╰────╯
   ╲
```
头微右转，尾巴左摆。

### 3.2 眨眼帧（随机触发）

```
   ╱╲    ╱╲
  ╱  ╲──╱  ╲
  │  ─    ─ │
  │    ω    │
   ╲      ╱
    ╰────╯│
           │
```
眼睛变为横线 `─`。插入闲置序列中随机触发（沿用现有 IDLE_SEQUENCE 机制）。

### 3.3 表情状态（触发式）

**开心** — 被抚摸时（/buddy pet）
```
   ╱╲    ╱╲
  ╱  ╲──╱  ╲
  │  ◠    ◠ │
  │   ╰╯   │
   ╲      ╱
    ╰────╯│
           │
  ♥ ♥    ♥ ♥      ← 爱心飘浮（2.5秒）
```

**惊讶** — 任务完成时
```
   ╱╲    ╱╲
  ╱  ╲──╱  ╲
  │  ◉    ◉ │
  │    ○   │
   ╲      ╱
    ╰────╯│
           │
```

**思考** — AI 处理中
```
   ╱╲    ╱╲
  ╱  ╲──╱  ╲
  │  ●    ● │
  │   ∪    │
   ╲      ╱
    ╰────╯ ╱
          ╱
```
嘴巴变为 ∪（张嘴），尾巴卷起。使用 ∪ 而非 ◡，因为 ∪ 在所有等宽字体中可正常渲染。

**困倦** — 长时间无操作
```
     z
   ╱╲    ╱╲
  ╱  ╲──╱  ╲
  │  ─    ─ │
  │    ω    │
   ╲      ╱
    ╰────╯│
           │
```

## 4. 交互行为

### 4.1 状态感知（自动）

| 用户操作 | 精灵状态 | 触发条件 |
|----------|----------|----------|
| AI 思考中 | 思考帧（尾巴卷起） | 消息发送后到响应开始 |
| 任务完成 | 惊讶帧 → 开心帧 | 工具调用完成、任务结束 |
| 长时间无操作 | 困倦帧 + z 字符 | 空闲超过 60 秒 |
| 用户输入时 | 闲置循环帧 | 默认状态 |

状态感知通过读取现有 `AppState` 中的 `companionReaction` 和会话状态实现，无需新增状态字段。

### 4.2 抚摸互动（/buddy pet）

- 触发开心帧 + 爱心飘浮动画（2.5秒）
- 爱心使用 `♥` 字符，颜色为 `#FF6900`
- 沿用现有 `companionPetAt` 时间戳机制

### 4.3 自动吐槽（语音气泡）

使用现有 `SpeechBubble` 组件，小米主题语录：

```
「正在优化中...」
「为发烧而生」
「Are you OK?」
「专注、极致、口碑、快」
「站在风口上，猪都能飞起来」
「小米，永远相信美好的事情即将发生」
```

触发条件与现有 buddy 系统相同（模型生成 companionReaction）。

## 5. 技术集成

### 5.1 集成方式

**替换策略：** 移除现有18种动物物种，小米猫成为 buddy 系统的唯一 companion。简化物种选择逻辑——所有用户自动获得小米猫，无需选择或抽取。

### 5.2 需要修改的文件

| 文件 | 修改内容 |
|------|----------|
| `src/buddy/types.ts` | 移除所有旧物种常量和 SPECIES 数组，仅保留 `xiaomi_cat`；移除稀有度系统（RARITIES、RARITY_WEIGHTS 等）；移除 EYES、HATS 数组（小米猫不支持自定义眼睛/帽子） |
| `src/buddy/sprites.ts` | 移除所有旧物种的 BODIES 数据，添加 `BODIES[xiaomi_cat]`（3帧闲置 + 眨眼帧 + 4帧表情）；简化 `renderSprite` 和 `renderFace`（仅处理 xiaomi_cat） |
| `src/buddy/companion.ts` | 简化 `roll` 函数——不再需要 PRNG、稀有度抽取、物种选择，直接返回固定的小米猫 bones；移除 `mulberry32`、`hashString`、`pick`、`rollRarity` 等函数 |
| `src/buddy/CompanionSprite.tsx` | 移除稀有度颜色映射，硬编码小米橙 `#FF6900`；移除 `{E}` 替换逻辑（眼睛固定为 `●`）；适配7行高度的精灵 |
| `src/buddy/useBuddyNotification.tsx` | 移除 teaser window 逻辑（不再需要预告） |
| `src/buddy/prompt.ts` | 更新 companion intro text 描述 |
| `src/utils/theme.ts` | 可选：添加 `xiaomi` 品牌色 token（`#FF6900`），或直接在 sprite 代码中硬编码 |

### 5.3 特殊处理

**高度问题：** 小米猫精灵高7行（耳朵+尾巴），比现有精灵高2行。需要：
- 检查 FullscreenLayout 的 bottom slot `maxHeight` 是否足够
- 检查非全屏模式下 sprite column 的高度是否影响布局
- 如果空间不足，可将耳朵压缩为单行（牺牲部分视觉效果）

**眼睛固定：** 小米猫的眼睛固定为 `●`，不使用 `{E}` 占位符。`renderSprite` 中跳过 `{E}` 替换。

**颜色固定：** 小米猫精灵的颜色固定为小米橙 `#FF6900`，不使用 `RARITY_COLORS` 映射。`CompanionSprite` 中硬编码颜色。

**伴侣系统简化：** 移除稀有度、物种选择、自定义眼睛/帽子等复杂度。`getCompanion()` 直接返回固定的小米猫 bones + 用户的 soul（name, personality）。

### 5.4 与现有系统的兼容性

- 复用现有 `CompanionSprite` 组件，简化内部逻辑
- 复用现有 `SpeechBubble` 组件
- 复用现有 `companionPetAt` 时间戳机制
- 复用现有 `AppState.companionReaction` 状态
- 复用现有 `/buddy` 命令系统（hatch、pet、mute 等）

### 5.5 可删除的代码

以下现有代码因不再需要而可安全删除：

- `src/buddy/types.ts` 中的：`duck`、`goose`、`blob` 等18个物种常量、`SPECIES` 数组、`EYES` 数组、`HATS` 数组、`RARITIES` 数组、`RARITY_WEIGHTS`、`RARITY_STARS`、`RARITY_COLORS`、`Species` 类型、`Eye` 类型、`Hat` 类型、`Rarity` 类型、`StatName` 类型、`STAT_NAMES`
- `src/buddy/sprites.ts` 中的：`BODIES` 对象中所有旧物种数据、`HAT_LINES` 对象
- `src/buddy/companion.ts` 中的：`mulberry32`、`hashString`、`pick`、`rollRarity`、`rollStats`、`rollFrom` 函数、`RARITY_FLOOR`、`SALT` 常量
